# GitHub Secrets Configuration

This document lists the secrets that need to be added to the GitHub repository settings for CI/CD to work properly.

## Required Secrets

Navigate to your repository → Settings → Secrets and variables → Actions → New repository secret

### 1. RENDER_API_KEY
**Description:** API key for Render deployment  
**How to get it:**
1. Log in to [Render Dashboard](https://dashboard.render.com/)
2. Go to Account Settings → API Keys
3. Create a new API key
4. Copy and paste the key value

**Value:** `rnd_xxxxxxxxxxxxxxxxxxxxx`

---

### 2. RENDER_SERVICE_ID
**Description:** The ID of your Render web service  
**How to get it:**
1. Go to your Render Dashboard
2. Select your web service
3. The service ID is in the URL: `https://dashboard.render.com/web/srv-XXXXXXXXXXXXX`
4. Copy the `srv-XXXXXXXXXXXXX` part

**Value:** `srv-xxxxxxxxxxxxxxxxxxxxx`

---

## Optional Secrets (for future enhancements)

### 3. SENTRY_DSN (Optional)
**Description:** Sentry error tracking DSN  
**Value:** `https://xxxxx@sentry.io/xxxxx`

### 4. SMTP_PASSWORD (Optional)
**Description:** Email service password for sending emails  
**Value:** Your SMTP password or app-specific password

---

## Setting Secrets

### Via GitHub UI:
1. Go to `https://github.com/kk121288/BTEC-backend/settings/secrets/actions`
2. Click "New repository secret"
3. Enter the name and value
4. Click "Add secret"

### Via GitHub CLI:
```bash
gh secret set RENDER_API_KEY --body "your-api-key-here"
gh secret set RENDER_SERVICE_ID --body "srv-your-service-id"
```

---

## Security Notes

- **Never commit secrets to the repository**
- Rotate API keys regularly
- Use different keys for staging and production
- Keep this document updated when adding new secrets
